Using ressources managers

Ressources managers

They are meant to hold file ressources, eg textures, fonts, etc.

You can easily create your own ressource holder, by inheriting from sgl::Ressource:

class MyRessource : public Ressource
{
public:
MyRessource() = default;
void load(const std::string& path)
{
// load m_data from given path
}
// this is needed for type erasure, so that the ressources can all share the same interface
void* fetch()
{
return static_cast<void*>(&m_data);
}
private:
MyType m_data;
};

How to use them

Say you want to create sf::Textures for a scene, and want to avoid having dozen of them defined in your class. In the scene's constructor you just have to do as follows:

MyScene::MyScene(int id) :
sgl::Scene(id)
{
// option 1
auto& res1 = m_ressources.get<sgl::TextureRessource>("texture.png");
my_sprite.setTexture(*static_cast<sf::Texture*>(res1.fetch()));
// option 2
// better, but you need to provide two types: the ressource and the inner type
auto& res2 = m_ressources.getAs<sgl::TextureRessource, sf::Texture>("texture.png");
my_sprite.setTexture(res2);
}
